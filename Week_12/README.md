Aplikasi Weather App dengan Firebase Authentication merupakan hasil penggabungan dua project Flutter yang dikembangkan menjadi satu aplikasi yang terintegrasi dan fungsional. Aplikasi ini berhasil mengimplementasikan sistem autentikasi pengguna menggunakan Firebase Authentication dengan metode email dan password, sehingga hanya pengguna yang telah terdaftar dan login yang dapat mengakses fitur informasi cuaca. Alur autentikasi dibangun menggunakan StreamBuilder yang memantau perubahan status login secara real-time, sehingga perpindahan halaman antara LoginPage dan WeatherPage terjadi secara otomatis tanpa memerlukan navigasi manual.

Pada sisi layanan cuaca, aplikasi memanfaatkan OpenWeatherMap API untuk mengambil data cuaca secara dinamis berdasarkan nama kota yang diinputkan pengguna. Data yang ditampilkan meliputi suhu saat ini, suhu terasa, kelembaban udara, kecepatan angin, jarak pandang, dan tekanan udara. Pengambilan data dilakukan secara asinkron menggunakan HTTP GET request, dan hasilnya diproses melalui model WeatherModel yang mengkonversi format JSON dari API menjadi objek Dart yang terstruktur.

Dari segi arsitektur, aplikasi ini menerapkan prinsip pemisahan tanggung jawab (separation of concerns) dengan membagi kode ke dalam beberapa lapisan yaitu Model, Service, dan Page. Setiap komponen memiliki fungsi yang spesifik dan independen sehingga kode menjadi lebih terorganisir, mudah dipahami, dan mudah dikembangkan di masa mendatang. Penerapan arsitektur ini juga memudahkan proses debugging karena setiap lapisan dapat diperiksa secara terpisah.

Secara keseluruhan, aplikasi ini telah berhasil dijalankan pada perangkat Android dan menunjukkan bahwa integrasi antara Firebase Authentication dan layanan API pihak ketiga dapat dilakukan dengan baik menggunakan Flutter. Proyek ini menjadi bukti bahwa Flutter mampu digunakan untuk membangun aplikasi mobile yang lengkap dengan fitur autentikasi dan pengambilan data dari internet secara bersamaan dalam satu project yang terstruktur.
